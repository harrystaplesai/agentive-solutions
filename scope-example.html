Project Overview

Project summary

01

Meridian Industrial Services manages equipment certifications, safety registrations, and compliance documentation across six regional business units. Staff members forward certificates to shared inboxes where they accumulate without classification, creating a growing backlog that makes retrieval during regulatory inspections slow and unreliable.

This project delivers an AI system that monitors those inboxes, extracts and classifies incoming documents, stores them in a searchable database, and provides a chatbot interface for instant retrieval. The system runs autonomously with human oversight, replacing 15+ hours of weekly manual sorting with automated processing that completes in seconds.

Process mapping and pain points

02

Current workflow

  1. Certificates arrive as email attachments to two shared inboxes
  2. Admin staff manually review each email and identify document type
  3. Documents sorted into equipment folders on shared drive (when capacity allows)
  4. Equipment IDs, expiration dates, and metadata entered into spreadsheet by hand
  5. Retrieval requires searching through folders or asking the person who filed it
  6. Expired certificates discovered only during audits or inspector requests

Pain points

15+ hours per week on manual sorting

Two staff members spend a combined 15 hours each week sorting, filing, and logging documents that arrive across both inboxes.

5 to 15 minute retrieval during inspections

When a regulator asks for a specific certificate, staff search through nested folders and email archives. Some documents take over 15 minutes to locate.

Expired certificates go undetected

No automated tracking for expiration dates. The team discovers gaps during audits, creating compliance exposure and operational risk.

Automation scope

  • Automated email monitoring and attachment extraction from both inboxes
  • AI classification by document type, equipment unit, and business region
  • Metadata extraction including equipment IDs and expiration dates
  • Vector database creation for semantic search across all documents
  • Chatbot interface for instant natural language and structured queries
  • Automated expiration alerts sent 60 days before certificate deadline

Automation outputs

  • Organized document library mirroring Meridian's equipment hierarchy
  • Searchable database with both semantic and structured query support
  • Chatbot that returns documents with direct download links and previews
  • Expiration tracking dashboard with proactive alert notifications
  • Audit log of all document processing and retrieval activity

Tech stack

03
Layer Technology Purpose
Email processing IMAP integration via N8N Automated monitoring and attachment extraction from Microsoft 365 mailboxes
AI / ML OpenAI GPT or Claude Document classification, metadata extraction, and chatbot reasoning
Database Supabase with pgvector Structured metadata storage and vector embeddings for semantic search
Orchestration N8N Workflow automation, scheduling, error handling, and retry logic

Interface Custom chatbot application Natural language and structured query interface for document retrieval

Security considerations

  • Encrypted document storage and transmission (TLS in transit, AES at rest)
  • Role based access control and user authentication
  • All AI API calls made server side with no client exposure of keys
  • Audit logging for all document access and retrieval events

Scalability

  • Cloud architecture supports unlimited document volume growth
  • Horizontal scaling of processing if email volume increases
  • Additional business units and document types added without rework
  • Multi region support for future office expansion
